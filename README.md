# Credit-Card-Fraud-Detection-using-ML-and-DL



https://credit-card-fraud-detection-using-ml-and-dlgit-mjvoogegnabw9hd.streamlit.app/


ML InnovateX Hackathon – Problem Statement

Theme: Credit Card Fraud Detection (ML + ANN + Deployment)

Dataset:
Use the Credit Card Fraud Detection dataset from Kaggle.
https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

Objective:
Design and build a robust, real-world fraud detection system using both Machine Learning models and an Artificial Neural Network (ANN).

Tasks:
1. Data Understanding & Cleaning
- Explore dataset structure, data types, and missing values
- Perform necessary cleaning and preprocessing
2. Exploratory Data Analysis (EDA)
- Perform univariate & bivariate analysis
- Identify fraud patterns and key insights
- Visualize important trends
3. Data Preprocessing
- Feature scaling
- Handle class imbalance
- Train-test split
4. Feature Engineering & Selection
- Create/transform meaningful features
- Apply at least one feature selection technique
- Justify your approach
5. Model Building (Machine Learning)
- Train at least 4 models
- Compare performance and select best model
- Perform Hyperparameter Tuning
6. ANN Model Development
- Design ANN architecture
- Specify layers, activation functions, optimizer, and learning rate
- Justify best configuration
7. Model Evaluation
- Precision, Recall, F1-Score, ROC-AUC
- Compare ML vs ANN
8. Hyperparameter Tuning & Cross Validation
- Use GridSearchCV / RandomizedSearchCV
- Apply k-Fold Cross Validation
9. Overfitting / Underfitting Analysis
- Compare training vs validation performance
- Suggest improvements
10. Final Model Justification
- Select best model based on performance, generalization, and usability

Mandatory Deployment Requirement:
Students must deploy the final model as a working application.
The system should:
- Accept user input (transaction details)
- Predict whether the transaction is Legitimate or Fraudulent

Submission Requirements:
- Code Notebook
- Model Comparison Summary
- Deployed Application Link (Mandatory)

Evaluation Criteria:
- Problem understanding
- Handling imbalanced data
- Model selection
- ANN design
- Deployment quality
